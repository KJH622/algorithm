# 웰컴 키트

온사이트 대회에서 참가자 **N명**에게 티셔츠 1장과 펜 1자루가 들어 있는 웰컴 키트를 나눠주려 한다.

주문 규칙은 다음과 같다.

- 티셔츠는 **S, M, L, XL, XXL, XXXL**의 6가지 사이즈가 있으며, **같은 사이즈끼리 T장 묶음**으로만 주문할 수 있다.

- 펜은 한 종류이며, **P자루 묶음**으로 주문하거나 **한 자루씩** 주문할 수 있다.

티셔츠는 남아도 되지만 **부족하면 안되고,** 신청한 사이즈대로 나눠줘야 한다.

펜은 **남거나 부족하면 안되며 정확히 N자루가 필요하다.**

다음 값을 구하라.

1. 티셔츠를 **T장씩 최소 몇 묶음** 주문해야 하는지

2. 펜을 **P자루씩 최대 몇 묶음** 주문할 수 있는지와, 그때 **한 자루씩 몇 개** 주문하는지

## 입력

- 첫째 줄에 정수 N이 주어진다.

- 둘째 줄에 순서대로 **S, M, L, XL, XXL, XXXL** 사이즈를 신청한 사람 수 6개가 주어진다.

- 셋째 줄에 정수 **T, P**가 공백으로 주어진다.

## 출력

- 첫째 줄에 **티셔츠 최소 묶음 수**를 출력한다.

- 둘째 줄에 **펜의 P자루 묶음 수**와 **한 자루로 주문할 개수**를 공백으로 구분해 출력한다.

## 입력 예시 1

```
23
3 1 4 1 5 9
5 7
```

## 출력 예시 1

```
7
3 2
```

## 풀이 아이디어

### 1. 문제를 수량/묶음 단위로 바꿔 생각하기

**A. 티셔츠**

- 사이즈별 신청자 수: `S, M, L, XL, XXL, XXXL` -> 총 6개 값

- 티셔츠는 **같은 사이즈끼리만 T장 묶음**으로 주문 가능

- **남아도 되고, 부족하면 안 됨** -> 각 사이즈별로 **올림** 묶음이 필요

즉, 각 사이즈의 신청자 수 `x`에 대해 필요한 묶음 수는 
$$\left\lceil \frac{x}{T} \right\rceil$$
이고, **전체 필요한 묶음 수**는 
$$\sum_{i=1}^{6} \left\lceil \frac{x_i}{T} \right\rceil$$
이 된다.

> **왜 올림인가?**
>
> 예를 들어, T = 5인데 M 사이즈 신청이 6명이면 1묶음(5장)으로는 부족하므로 2묶음(10장)을 사야한다.
>
> 즉, 나눗셈 결과를 반드시 올림해야 한다.

**정수 연산으로 올림 구현하는 트릭**

파이썬/정수연산에서 올림은 `ceil(x/T) = (x + T - 1) // T` 로 계산한다.

- 예: x = 6, T = 5 -> (6+4)//5 = 2

- 예: x = 5, T = 5 -> (5+4)//5 = 1

- 예: x = 0, T = 5 -> (0+4)//5 = 0 (0명 신청이면 0묶음)

이 연산을 6개 사이즈에 적용해서 **합**하면 끝

**B. 펜**

- 펜은 **P자루 묶음** 또는 **한 자루씩** 주문 가능

- **정확히 N자루** 필요 (남거나 부족하면 안 됨)

가장 많은 수의 묶음을 쓰고, 남는 것을 낱개로 채우면 **항상 비용/묶음 수 면에서 최적**이다.

- **묶음 수** = `N // P` (P자루씩 최대 몇 묶음을 쓸 수 있는가?)

- **낱개 수** = `N % P` (묶음으로 채우고 남은 자루 수)

> **왜 이것이 항상 최적인가?**
>
> 묶음 수를 더 줄이고 낱개를 늘리는 것은 필요 이상으로 낱개 주문을 늘리는 것 뿐이라 비효율적이고, 묶음을 더 늘리면 총 자루 수가 N을 초과하므로 불가능하다.
>
> 따라서 `몫/나머지` 분해가 유일하고 최적이다.

### 2. 예시로 검증하기

**입력**

```
N = 23
sizes = [3, 1, 4, 1, 5, 9]
T = 5, P = 7
```

- 티셔츠(사이즈별 올림 묶음):

    - 3 -> (3 + 4) // 5 = 1

    - 1 -> (1 + 4) // 5 = 1

    - 4 -> (4 + 4) // 5 = 1

    - 1 -> (1 + 4) // 5 = 1

    - 5 -> (5 + 4) // 5 = 1

    - 9 -> (9 + 4) // 5 = 2

    **합 = 7묶음**

- **펜**

    - 묶음 수 = 23 // 7 = 3

    - 낱개 수 = 23 % 7 = 2

    **-> 7자루 묶음 3개 + 낱개 2개** 

### 3. 자주 하는 실수 & 주의점

- 티셔츠에서 **총합(N)으로 묶음 계산**하면 안 됨.

    사이즈별로 따로 묶음 계산 후 **합산**해야 한다. (사이즈끼리 섞어 주문 불가)

- 올림을 직접 `math.ceil(x / T)` 로 해도 되지만 정수 트릭 `(x + T - 1) // T` 가 안전하고 빠름

- 펜은 **정확히 N자루**가 맞아야 하므로 `N // P, N % P`를 그대로 쓰면 된다.

### 4. 복잡도

- 입력 처리 + 6개 사이즈 계산 -> **O(1)**

- 매우 빠르고, N 크기와 무관

### 5. 한 줄 아이디어 요약

- **티셔츠** : 사이즈별로 `ceil(신청자/T)`을 구해 모두 더한다. (정수식: `(x + T - 1) // T`)

- **펜**: `N = (N // P) * P + (N % P)` 구조로 묶음과 낱개를 바로 구한다.

## 파이썬 코드

### 1번째 방식

```
# 웰컴 키트

# 티셔츠는 S, M, L, XL, XXL, XXXL 6가지 사이즈
# 티셔츠는 같은 사이즈의 T장 묶음으로만 주문 가능
# 펜은 한 종류로, P자루씩 묶음으로 주문하거나 한 자루씩 주문 가능
# 티셔츠는 남아도 되지만 부족해서는 안되고 신청한 사이즈대로 나누어주어야 한다.
# 펜은 남거나 부족해서는 안되고 정확히 참가자 수만큼 준비되어야 한다.

# 티셔츠를 T장씩 최소 몇 묶음 주문해야 하는가.
# 펜은 P자루씩 최대 몇 묶음 주문할 수 있고 펜을 한 자루씩 몇 개 주문해야 하는가.

n = int(input()) # 참가자 수

N_P = list(map(int, input().split()))
T, P = map(int, input().split())

# 티셔츠: 각 사이즈별로 올림 나눗셈 묶음 수 환산
shirt = sum((x + T - 1) // T for x in N_P)

# 펜: P자루 묶음 최대 개수와 남는 낱개
pen_bundles = n // P
pen_singles = n % P

print(shirt)
print(pen_bundles, pen_singles)
```

### 2번째 방식

```
N = int(input())
T_N_input = list(map(int, input().split()))
T_P_input = list(map(int, input().split()))

sum = 0
# 티셔츠
for i in T_N_input:
    if(i % T_P_input[0] == 0):
        sum = sum + (i//T_P_input[0])
    else:
        if(i < T_P_input[0]):
            sum = sum + 1
        else:
            sum = sum + (i//T_P_input[0]) + 1

# 펜
P_set = N//T_P_input[1]
P_each = N%T_P_input[1]

print(f"{sum}")
print(f"{P_set} {P_each}")
```
